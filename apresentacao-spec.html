Conceito

O que significa ser spec driven

Use este slide para definir o conceito com sua linguagem. A recomendação aqui é combinar definição curta, contraste com o modelo tradicional e um benefício percebido rapidamente pela audiência.

Boas peças para incluir

  • Uma frase-manifesto que guie o restante do deck
  • Um antes/depois da rotina de um time
  • Uma imagem simples do fluxo spec → plan → tasks → code

Framework

Quatro blocos para operar com clareza

Intento

Qual mudança queremos produzir e para quem.

Contrato

Regras, critérios de aceite e evidências de sucesso.

Entrega

Como transformar especificação em tarefas, prompts e código.

1 Defina o problema com linguagem observável.
2 Prenda as decisões em artefatos reutilizáveis.
3 Deixe a implementação herdar o contexto, não inventá-lo.

Provocação

Se a especificação é fraca, a velocidade da AI apenas acelera a direção errada.
